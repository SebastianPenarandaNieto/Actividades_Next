# Triki (Tic-Tac-Toe) con Next.js y Tailwind CSS

Este es mi proyecto del clásico juego **Triki (3 en línea)**, desarrollado con **Next.js (App Router)** y **Tailwind CSS**.

---

## Reflexiones

1. **¿Qué ventajas tiene usar React frente a trabajar solo con JavaScript puro para interfaces web?** 
   React facilita mucho la creación de interfaces porque todo se maneja con componentes reutilizables. También actualiza la vista automáticamente cuando cambian los datos, cosa que con JavaScript puro hay que hacer manualmente.

2. **¿Qué beneficios ofrece Next.js al trabajar sobre React?** 
   Next.js agrega funciones importantes que React no trae solo, como rutas automáticas, renderizado en servidor, optimización de imágenes y mejor rendimiento. Eso lo hace más práctico para proyectos reales.

3. **¿Qué significa que Next.js tenga un App Router y por qué se recomienda usarlo?** 
   El App Router convierte la estructura de carpetas dentro de `app/` en rutas automáticamente. Se recomienda porque es la forma más actual de organizar proyectos en Next.js y ayuda a mantener todo más limpio.

4. **¿Qué diferencia hay entre client components y server components en Next.js?** 
   - Los **client components** corren en el navegador y se usan cuando hay interactividad (como clics o estados).
   - Los **server components** se renderizan en el servidor antes de mandarse al navegador, lo que mejora el rendimiento y carga más rápido.

5. **¿Por qué crees que en desarrollo profesional se usan repositorios públicos o privados en GitHub en lugar de solo trabajar en carpetas locales?** 
   Porque en GitHub el proyecto queda respaldado, se puede trabajar en equipo y llevar un control de cambios. En cambio, en carpetas locales se corre el riesgo de perder el trabajo y no hay historial de versiones.
